# 我写了一个会自己长大的 AI 记忆系统，然后把它开源了

> 三个月前我在想一个问题：为什么 AI Agent 每次都要重新理解我的代码？于是我动手做了一个系统，后来发现它不光能记住代码，还能自己进化。现在开源了。

## 起因：AI 记性太差了

事情是这样的。

我用 Claude Code 写代码的时候发现一个很烦的事情——每次新开一个会话，它就跟失忆了一样。明明昨天才跟它折腾了两个小时的认证模块，今天问一句"我们的支付流程怎么接的"，它又开始从头读代码。

读就读吧，一读就是 47 个文件，2000 行代码，token 哗哗地烧。

我当时就想：能不能把代码预先"消化"一遍，让 AI 直接查索引，而不是每次都去读源文件？

于是我开始折腾。

---

## 第一版：就是个代码索引器

一开始的想法很简单——用正则把代码里的函数、类、import 关系提取出来，存到 SQLite 里，加个 FTS5 全文搜索。AI 问"authenticateUser 在哪"的时候，直接查索引返回，不用读文件。

写完之后试了一下，效果出乎意料的好。

正好那时候 CodeGraph 开源了，它的 benchmark 数据验证了我的想法：**有预索引的情况下，AI 理解代码的工具调用减少 58%，token 消耗降低 47%，整体便宜 16%。**

但我很快发现一个问题：这只是"代码的记忆"，不是"我的记忆"。

---

## 转折：代码只记了一半

有一天我跟 AI 聊天，它建议我用 yarn 装个包。

我说：我不喜欢 yarn，我用 pnpm。

它回：好的，已记录。然后下次对话，它又建议我用 yarn。

我突然意识到——代码索引只解决了"项目是什么"，但完全没有解决"我是谁"这个问题。我的偏好、习惯、踩过的坑、做过的决策，这些东西散落在每次对话里，从来没有被系统地记下来过。

于是我加了个东西：**用户画像记忆体。**

---

## 核心设计：双层记忆

现在的 Muti-MemoAgent 有两个层次：

**L1 — 本地 SQLite（热记忆）**

所有记忆先在本地存一份。FTS5 全文搜索、向量语义搜索、符号精确搜索全在本地跑。查询一个函数 0.5 毫秒返回，零网络延迟，完全离线可用。

**L2 — Xiami 云端（冷记忆）**

异步同步到云端，走 Neo4j 图谱存储。不同项目的记忆在云端互相发现关联——项目 A 的认证模块和项目 B 的认证模块是同一套模式？自动标记。你的偏好"喜欢 pnpm"和项目里的 `pnpm-lock.yaml`？自动关联。

这样设计的原因是：本地快，云端广，两者互补。断网也能用，联网更强大。

---

## 最让我兴奋的功能：自进化

这个是后来加的，但成了最有意思的部分。

传统的记忆系统就是个数据库——存进去，查出来，仅此而已。但人的记忆不是这样的。人的记忆会遗忘、会重组、会把相关的概念联系起来。

所以我在系统里加了个**进化引擎**。

每 24 小时，系统会做一次自我评估。评估四个维度：记忆质量、使用效用、进化活跃度、协作贡献。得分低的记忆体怎么办？

不是删掉，是**变异**。

- **拆分：** 一个项目记忆体太臃肿了？按标签自动拆成 API、数据库、前端三个。
- **合并：** 两个记忆体高度重叠？合并成一个，去重。
- **巩固：** 短期工作记忆里 3 条都跟"部署流程"相关？合并成一条长期记忆。
- **竞争：** 两个 Agent 覆盖同一领域？用真实查询比赛，胜者留下，败者知识迁移。

运行了一个多月，一个原本 1 个 Agent 的项目，自动进化成了 4 个专业化记忆体。完全没有手动干预。

---

## 遗忘引擎：不是越存越多

还有个反直觉的设计：记忆系统应该**主动遗忘**。

很多 AI 记忆方案的问题是：存进去就不管了。三个月后，8000 条记忆里 6000 条是垃圾。

我在里面加了时间衰减 + 重要性评分 + 梦境巩固。

每晚跑一次"梦境"周期：低分记忆蒸馏成摘要后删除原文，高分记忆被强化。运行一个月后，记忆总量反而下降了 60%，但搜索命中率提升了 40%。

**记住该记的，忘掉该忘的——这才是好记忆。**

## 快速上手

开源地址：https://github.com/YPing-A/Muti-MemoAgent

```bash
# 克隆
git clone https://github.com/YPing-A/Muti-MemoAgent.git
cd Muti-MemoAgent && pnpm install

# 首次会自动打开浏览器 → 注册 Xiami → 获取密钥
# 然后初始化
npx memograph init --xiami-key xiami_sk_xxx

# 日常就这么用
memograph index              # 索引代码
memograph analyze             # 分析架构
memograph search "支付流程"    # 搜索记忆
memograph watch               # 保存即同步
memograph dashboard           # 可视化图谱
```

## 写在最后

这个项目的核心思路其实很简单：**AI 需要记忆，但不是随便堆数据的那种，而是像人一样会整理、关联、遗忘、进化的记忆。**

CodeGraph（43K star）证明了预索引能省钱，Understand-Anything（54K star）证明了代码可视化有价值，Xiami 提供了云端持久化。我把这三者揉在一起，加上了自进化引擎，就有了 Muti-MemoAgent。

目前 13 个包、30000+ 行 TypeScript、20+ 语言支持，已通过完整的功能测试。
